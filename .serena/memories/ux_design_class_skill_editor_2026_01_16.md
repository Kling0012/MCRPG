## 問1: ユーザー導入設計（Onboarding）

### 1.1 ファーストラン体験（5分フロー）

```
STEP1: ウェルカム画面（30秒）
  - RPGプラグインへようこそ！
  - クイックスタート or テンプレートから選択

STEP2: テンプレート選択（45秒）
  - プリセット: 戦士/魔法使い/弓使い/大盾使い
  - 💡 テンプレートは後で変更可能

STEP3: 最小限の入力（2分）
  - 必須項目のみ: id, name, displayName
  - □ 詳細設定を後で行う（推奨）

STEP4: プレビュー＆調整（1分）
  - アイコン、表示名、説明を確認
  - 必要に応じて編集

STEP5: 成功フィードバック（30秒）
  - 🎉 おめでとうございます！
  - 続けて学ぶ or 終了
```

### 1.2 チュートリアル形式

| 形式 | メリット | 採用判断 |
|------|---------|---------|
| 対話式ツアー | 手を動かしながら学べる | ✅ 簡易版を採用 |
| インタラクティブガイド | 自己ペースで進められる | ✅ メイン方式 |
| テキストWiki | 検索性が高い | ✅ リファレンス用 |

### 1.3 AIパーソナライズ（2025トレンド）
- ユーザーの操作履歴からおすすめテンプレートを提案
- 作成した職業の傾向から新しいスキルをサジェスト

## 問2: 情報の階層化・簡略化設計

### 2.1 プログレッシブ開示（Progressive Disclosure）

```
LAYER 1: 必須（Required）- 3項目
  1. id          - 一意識別子
  2. name        - 内部名
  3. displayName - 表示名

LAYER 2: 基本（Basic）- 5項目
  4. description    - 説明文
  5. icon           - アイコン
  6. maxLevel       - 最大レベル
  7. type           - 種類（Skillのみ）
  8. rank           - ランク（RPGClassのみ）

LAYER 3: 推奨（Recommended）- 4項目
  9.  statGrowth     - ステータス成長
  10. damage         - ダメージ設定
  11. skillTree      - スキルツリー設定
  12. availableClasses - 利用可能クラス

LAYER 4: 上級（Advanced）- 残り全て
  13. variables          - カスタム変数
  14. formulaDamage      - 数式ダメージ
  15. targeting          - ターゲット設定
  ...（他高度な設定）
```

### 2.2 UIパターン

| パターン | 使用箇所 | 実装例 |
|---------|---------|--------|
| アコーディオン | LAYER 3/4の各セクション | `▼ ステータス成長` をクリックで展開 |
| タブ切り替え | 関連する設定グループ | `[基本] [ステータス] [スキル] [詳細]` |
| プログレス的披露 | 詳細機能のアンロック | 「基本設定完了後に詳細設定が解放」 |
| ツールチップ | 用語の説明 | `MP回復量` にホバーで説明表示 |

### 2.3 用語の平易化マッピング

| 専門用語 | 平易化表現 | 説明文例 |
|---------|-----------|---------|
| StatGrowth | ステータス成長 | レベルアップ時に上昇する能力値 |
| ExpDiminish | 経験値減衰 | 高レベルになるほど経験値取得量が減る |
| LevelDependentParameter | レベル変動値 | スキルレベルに応じて変化する数値 |
| SkillTreeConfig | スキルツリー | スキルの習得ルート図 |
| TargetingConfig | 範囲設定 | 誰に効果を与えるかの指定 |
| ComponentEffect | エフェクト構成 | スキル発動時の詳細な挙動 |

## 問3: エラー体験設計

### 3.1 バリデーションタイミング（NN/G指針）

| タイミング | バリデーション内容 | フィードバック |
|-----------|------------------|---------------|
| 入力中 | フォーマットチェック | 入力欄の下にヒント表示 |
| フォーカスアウト | 必須チェック、重複チェック | ✓または⚠️アイコン表示 |
| 保存時 | 全項目の整合性チェック | エラー箇所をハイライト |
| 保存後 | 依存関係のチェック | 関連する他ファイルも警告 |

### 3.2 エラーメッセージデザイン原則

1. **明確な問題指摘**: 「IDが空です」ではなく「一意識別子が必要です」
2. **具体的な解決策**: 次に何をすべきかを明示
3. **視覚的な位置づけ**: エラー箇所をアイコンと色でハイライト
4. **優先順位の表示**: 致命的エラーと警告を区別

### 3.3 Auto-Fix戦略

| エラー種類 | Auto-Fix内容 | ユーザー確認 |
|-----------|-------------|-------------|
| 空の必須項目 | デフォルト値/サジェスト値を挿入 | 要 |
| 無効なカラーコード | 最も近い有効コードに変換 | 要 |
| 数値範囲外 | 最小/最大値にクランプ | 不要（通知のみ） |
| ID重複 | サフィックス付与（例: warrior_2） | 要 |
| フォーマット誤り | 自動修正（可能な場合） | 不要（通知のみ） |

### 3.4 複数エラーの表示

- エラーの数を冒頭で表示
- 重大度順に並べ替え
- 「すべて自動修正」ボタンを提供
- 各エラーに個別の修正アクション

## 問4: 成功体験設計

### 4.1 5分クイック作成フロー

```
0:00 - 0:30: STEP1 テンプレート選択
0:30 - 1:30: STEP2 基本情報入力
1:30 - 2:30: STEP3 簡易設定（スライダー）
2:30 - 4:00: STEP4 スキル追加（テンプレートから）
4:00 - 5:00: STEP5 プレビュー＆保存
```

### 4.2 達成感フィードバック（ゲーミフィケーション）

| タイミング | フィードバック内容 | 実装方法 |
|-----------|------------------|---------|
| 作成完了 | 花火エフェクト+サウンド | パーティクル効果 |
| 保存時 | `✓ 保存しました` トースト | Bukkitの通知 |
| マイルストーン | `🏆 5個目の職業を作成！` | 実績システム |
| 初回機能 | `🌟 新機能「詳細設定」が解放！` | プログレス的披露 |

### 4.3 進捗表示

- ステップの進捗を視覚的に表示
- 完了したステップにチェックマーク
- 現在のステップをハイライト
- 残り時間の見積もり
